Inline vs inline blocks, what does it mean?

by Tiff Wang

When building a website, understanding different display properties is crucial for creating clean and well-structured layouts. "Inline," "Block," and "Inline-block" are three commonly used display values, but I sometimes found them confusing when attempting to apply them in my web development. In this blog, I'll take an in-depth look at these display types with examples of code and screenshots. If you've ever wondered about these concepts like me, you've come to the right place!

display: block

Before looking into the "Inline" and "Inline-block", let's have a look what is "Block". I first created 4 divs with different colors without setting the width, and you can see that it lines vertically with each div takes the whole width of the viewport.
This shows that the divs are display: block. Even I change the width to shorten each div, they will not move up to stay side by side, but still lines vertically.
Therefore, when an element is set to display: block;, it will create a rectangular box that spans the entire width of its parent container by default. This means that block-level elements start on a new line and occupy the full width available.

Here are some elements that display block by defalt:

  • div
  • section
  • p
  • h1
  • li
  • br
  • hr

display: inline

Now let's see what will happen if we add a display:inline.
You can see that under an inline display, the line breaks disappear, as well as the width of each div. This means that when applying display:inline, multiple inline elements will appear next to each other horizontally, as long as there's enough space, any height and width properties will no longer have effect, where the size of the div depends on its content inside.
Let's demenstrate it by increasing the font of the content or adding more texts.

Here are some elements that display inline by defalt:

  • a
  • span
  • img

display: inline-block

We first change the edited content back. Now let's see what will happen if we change the display to display:inline-block.
We can see that the divs appear next to each other horizontally while maintaining the width setting.

What if we add heights to the divs?
Or setting different width and height for each div?
It can be seen that unlike inline elements, inline-block elements have combines the characteristics of both inline and block elements. They appear next to each other horizontally, just like inline elements, but at the same time maintain its specific width and height values like block elements.

Based on its charastestic, inline-block is expecially useful when you want to control the layout of elements while maintaining their inline-like formatting.

Conclusion

  • block: The block element always starts on a new line and takes the full width of the viewport width or its parent's width. You can set width and height value to block elements.
  • inline: Inline elements appears next to each other and only takes the width depends on its content inside. You can not set the width or height value for inline elements.
  • inline-block: Inline-block element also appears next each other. But unlike inline, you are able to set width and height values for inline-block element.
